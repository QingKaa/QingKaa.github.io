vite本地开发开启https

清咖
2024-06-17
浏览器

@vitejs/plugin-basic-ssl

参照vite的配置选项server-httpsopen in new window，这里提供了一种方法，使用@vitejs/plugin-basic-ssl插件。

安装依赖依赖

pnpm i @vitejs/plugin-basic-ssl

vite配置使用该插件

import { defineConfig } from "vite";
import basicSsl from "@vitejs/plugin-basic-ssl"
// ...

export default defineConfig({
    plugins:[
        basicSsl()
    ]
    // ...
})

重启服务即可开启

优点

最大的优点就是使用是否简单，只需要安装一个插件，然后配置一下即可。

弊端

  1. 默认生成的证书是自签名的，所以浏览器会提示不安全。
  2. 除了浏览器之外的一些环境可，如微信小程序，无法顺利进行开发。

使用 mkcert 生成安全的证书

mkcertopen in new window

这是是一个非常简单的工具，可以零配置制作本地信任的开发证书。

打开官网，可以直接下载安装包安装或者按说明文档进行安装：

下载安装包

在Release 中找到你需要的版本进行下载：

使用scoop安装

(本人使用这种方式，没有优略之说)使用scoop安装需要先安装 scoop ：

irm get.scoop.sh | iex

完了后，使用scoop安装mkcert：

scoop bucket add extras
scoop install mkcert
mkcert -install

安装完成后会弹出一个框，确定即可：

最后创建证书：

mkcert localhost 127.0.0.1 ::1

查看证书的位置在C:\Users\用户名下：

配置vite

可以将证书复制到项目根目录下，然后配置vite的 https 选项即可。
有证书了就不需要使用 basicSsl 插件了，直接使用 vite 的 https 选项即可。配置如下：

import fs from "node:fs";
import { defineConfig } from "vite";
import basicSsl from "@vitejs/plugin-basic-ssl"
// ...

export default defineConfig({
    plugins:[
        // basicSsl()
    ],
    server:{
       https:{
          key: fs.readFileSync("certs/localhost+2-key.pem"),
          cert: fs.readFileSync("certs/localhost+2.pem"),
       }
    }
    // ...
})

重启服务即可开启https。
