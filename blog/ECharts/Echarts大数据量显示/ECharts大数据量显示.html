基本情况

以下均基于 "echarts": "^5.5.0" 版本。加载时间会与设备有关。

面对大数据量时，ECharts默认的显示效果会很差:

  1. 加载时间长。
  2. 页面卡顿。
  3. tooltip 显示非常慢。

十万条数据测试直接渲染大概需要 3 ~ 5s，tooltip 显示简直无法直视。

使用 dataZoom

dataZoom 组件常用于区域缩放，只显示一部分数据。

使用了dataZoom的情况下渲染基本上都是毫秒级，tooltip 显示也是是否流畅（前提是不能同时显示太多数据）

使用dataZoom可以有效的优化以上问题，不过需要注意的是，当数据量十分大的时候需要限制 dataZoom 的范围(maxSpan)，不然随着显示区域的增大，也会变得非常卡顿。

 let options: EChartsOption = {
    ...,
    dataZoom: {
        type: 'slider', // slider | inside
        xAxisIndex: 0,
        start: 0,
        end: 1,
        minSpan: 1, // 限制最小显示范围
        maxSpan: 5 // 限制最大显示范围
    }
 }

合理的配置可以极大的提升流畅性。关于 dataZoom 的配置可以参考 官方文档open in new window。

降采样策略 sampling: 'lttb'

折线图在数据量远大于像素点时候的降采样策略，开启后可以有效的优化图表的绘制效率，默认关闭，也就是全部绘制不过滤数据点。
可选：

'lttb' 采用 Largest-Triangle-Three-Bucket 算法，可以最大程度保证采样后线条的趋势，形状和极值。
'average' 取过滤点的平均值
'min' 取过滤点的最小值
'max' 取过滤点的最大值
'minmax' 取过滤点绝对值的最大极值 (从 v5.5.0 开始支持)
'sum' 取过滤点的和
文档说明open in new window

用了降采样策略的情况下由渲染到肉眼可见也基本是秒开，只是tooltip则惨不忍睹(丢失严重甚至无法显示出来)。

const options: EChartsOption ={
    series: [
            {
                // ...
                type: 'line',
                smooth: true,
                sampling: 'lttb',//
            }
        ]
      }

large 与 largeThreshold

是否开启大数据量优化，在数据图形特别多而出现卡顿时候可以开启。
开启后配合 largeThreshold 在数据量大于指定阈值的时候对绘制进行优化。
缺点：优化后不能自定义设置单个数据项的样式。
官方文档open in new window

特别注意，large 只能在 bar, scatter, candlestick, lines 使用，其他图表不支持。

不过效果是真的好，非常丝滑。渲染速度毫秒级，tooltip 显示正常，也不卡顿。

const options: EChartsOption ={
    // ...
    series: [
        {
            // ...
            data: [...seriesData],
            type: 'bar',
            large: true
        }
    ]
}

echartsInstance.appendData

略过~~

这是一个echarts实例的方法，用于分片加载数据。不是一次性加载数据，而是分批加载，在把加载的数据合并到已经渲染的图里面去，渲染新加入的数据块时不会清除原有已经渲染的部分。

此接口用于，在大数据量（百万以上）的渲染场景，分片加载数据和增量渲染。
注意：

现在不支持 系列（series） 使用 dataset 同时使用 appendData，只支持系列使用自己的 series.data 时使用 appendData。 目前并非所有的图表都支持分片加载时的增量渲染。目前支持的图有：ECharts 基础版本的 散点图（scatter） 和 线图（lines）。ECharts GL 的 散点图（scatterGL）、线图> （linesGL） 和 可视化建筑群（polygons3D）。
